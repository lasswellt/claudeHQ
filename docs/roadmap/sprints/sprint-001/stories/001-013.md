# 001-013: Hub Fastify server setup

## Context
The Hub is a Fastify HTTP server that hosts the REST API, WebSocket endpoints, static dashboard files, and health check. This story sets up the server with all plugins.

## Requirements
1. Create `packages/hub/src/server.ts` — main Fastify server setup
2. Register `@fastify/websocket` for WebSocket support
3. Register `@fastify/static` to serve dashboard static files (placeholder dir for now)
4. Register `@fastify/cors` if needed (same-origin should work)
5. Configure pino logging (built into Fastify)
6. Create `GET /health` route returning `{ status: 'ok', version, uptime }`
7. Load config from environment + config file using HubConfig schema
8. Graceful shutdown on SIGTERM/SIGINT
9. SPA fallback: `setNotFoundHandler` returns index.html for non-API GET requests

## Acceptance Criteria
- [ ] `node packages/hub/dist/server.js` starts Fastify on configured port
- [ ] `GET /health` returns 200 with `{ status: 'ok' }`
- [ ] WebSocket upgrade works at `/ws/agent` (empty handler for now)
- [ ] WebSocket upgrade works at `/ws/dashboard` (empty handler for now)
- [ ] pino logs show request/response in structured JSON
- [ ] SIGTERM triggers graceful shutdown (close server, exit 0)
- [ ] Config loaded and validated with HubConfig schema from @chq/shared

## Technical Notes
- Fastify 5.x required for @fastify/websocket v11.x
- `@fastify/websocket`: `fastify.get('/ws/agent', { websocket: true }, (socket, req) => {...})`
- Handler receives raw WebSocket object (not stream) in v11.x
- `@fastify/static`: `root: path.join(__dirname, '../dashboard-static'), wildcard: false`
- SPA fallback: only for GET requests not matching /api/*, /ws/*, /hooks/*
- Use `fastify.setErrorHandler()` for consistent error responses
