# 001-005: ESLint + Prettier configuration

## Context
Consistent code formatting and linting across the monorepo. The post-edit hooks in `.claude/hooks/` already reference formatting and linting — this provides the tooling they depend on.

## Requirements
1. Install ESLint 9 with flat config
2. Install `@typescript-eslint/parser` and `@typescript-eslint/eslint-plugin`
3. Create `eslint.config.js` at root with TypeScript strict rules
4. Install Prettier and create `.prettierrc`
5. Add `lint` and `format` scripts to root package.json
6. Ensure `.claude/hooks/post-edit-format.sh` and `post-edit-lint.sh` work

## Acceptance Criteria
- [ ] `pnpm lint` runs without errors on clean codebase
- [ ] `pnpm format` formats all files consistently
- [ ] ESLint catches `any` types (per project rules: no `any`)
- [ ] Prettier handles `.ts`, `.vue`, `.json`, `.md` files
- [ ] `.claude/hooks/post-edit-lint.sh` exits 0 on clean code

## Technical Notes
- ESLint 9 flat config: `eslint.config.js` with `export default [...]`
- Include `@typescript-eslint/recommended-type-checked` for strict rules
- Prettier config: `{ "singleQuote": true, "trailingComma": "all", "semi": true, "printWidth": 100 }`
- Ignore patterns: `node_modules`, `dist`, `.output`, `.nuxt`, `.turbo`

## Files Likely Affected
- `eslint.config.js`
- `.prettierrc`
- `.prettierignore`
- `package.json` (root — scripts + devDependencies)
