# 001-006: Docker infrastructure

## Context
Claude HQ runs as a self-contained Docker Compose app. The Hub serves the dashboard static files. This story creates the Dockerfiles and compose config so `docker compose up` works from Sprint 001.

## Requirements
1. Create `Dockerfile.hub` — multi-stage build using `turbo prune --docker`
2. Create `Dockerfile.agent` — multi-stage build for agent package
3. Create `docker-compose.yml` — Hub service with health check, volumes, env
4. Create `docker-compose.simple.yml` — simplified version (no Tailscale sidecar)
5. Create `.dockerignore`
6. Create `.env.example`
7. Create `Makefile` with build/up/down/logs targets

## Acceptance Criteria
- [ ] `docker compose build` succeeds
- [ ] `docker compose up -d` starts Hub on port 7700
- [ ] `curl http://localhost:7700/health` returns 200 (after Hub code exists)
- [ ] SQLite volume persists at `./data/db/`
- [ ] Recordings volume at `./data/recordings/`
- [ ] `.dockerignore` excludes node_modules, .git, docs, .claude

## Technical Notes
- Base image: `node:22-bookworm-slim` (NOT Alpine — better-sqlite3 needs glibc)
- Multi-stage: pruner → deps → builder → prod-deps → runtime
- Use `--mount=type=cache,id=pnpm,target=/root/.local/share/pnpm/store`
- Hub serves dashboard static from `packages/hub/dashboard-static/`
- HEALTHCHECK uses `node -e "fetch('http://localhost:7700/health')..."`
- See `docs/_research/2026-03-15_self-hosted-docker-deployment.md` for complete Dockerfiles
