# 001-002: TypeScript configuration with strict mode

## Context
Shared TypeScript configuration ensures consistent compilation settings across all packages. Strict mode is required by project rules.

## Requirements
1. Create `tsconfig.base.json` at root with strict settings
2. Create `packages/shared/tsconfig.json` extending base
3. Create `packages/agent/tsconfig.json` extending base with Node.js settings
4. Create `packages/hub/tsconfig.json` extending base with Node.js settings
5. Create `packages/dashboard/tsconfig.json` (Nuxt will manage this, but base extension)
6. Configure path aliases for `@chq/shared` if needed
7. Use `tsup` or plain `tsc` for building shared and backend packages

## Acceptance Criteria
- [ ] `strict: true` in base config
- [ ] `noUncheckedIndexedAccess: true` enabled
- [ ] All packages compile without errors via `tsc --noEmit`
- [ ] Path aliases resolve `@chq/shared` in agent and hub
- [ ] `target: ES2022`, `module: Node16` for backend packages

## Technical Notes
- Base config: `strict: true, noUncheckedIndexedAccess: true, esModuleInterop: true, skipLibCheck: true, forceConsistentCasingInFileNames: true`
- Backend packages: `target: ES2022, module: Node16, moduleResolution: Node16`
- Dashboard: Nuxt generates its own tsconfig; just extend base for shared settings
- Use `references` in root tsconfig for project references if desired, or keep simple with independent configs

## Files Likely Affected
- `tsconfig.base.json`
- `packages/shared/tsconfig.json`
- `packages/agent/tsconfig.json`
- `packages/hub/tsconfig.json`
- `packages/dashboard/tsconfig.json`
