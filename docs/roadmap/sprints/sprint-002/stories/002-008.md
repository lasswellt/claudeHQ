# 002-008: Hub hook receiver routes

## Context
Claude Code HTTP hooks POST JSON to the Hub when events occur (Stop, PreToolUse, PostToolUse, SubagentStop). The Hub stores these events and can forward them to dashboard clients.

## Requirements
1. `POST /hooks/stop` — log event, update session status
2. `POST /hooks/pre-tool-use` — log event, forward to dashboard
3. `POST /hooks/post-tool-use` — log event, forward to dashboard
4. `POST /hooks/subagent-stop` — log event
5. Store all events in `session_events` table
6. Correlate events to sessions via session_id in the hook payload

## Acceptance Criteria
- [ ] Each hook route accepts POST with JSON body
- [ ] Events stored in `session_events` table with payload
- [ ] `Stop` hook updates session status to completed/idle
- [ ] Events forwarded to dashboard subscribers
- [ ] Invalid payloads return 400 (not 500)

## Files Likely Affected
- `packages/hub/src/routes/hooks.ts` (new)
