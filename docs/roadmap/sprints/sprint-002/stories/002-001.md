# 002-001: Agent WebSocket client with reconnection

## Context
The agent daemon maintains a persistent WebSocket connection to the Hub. It must auto-reconnect with exponential backoff when disconnected, and handle all Hubâ†’Agent message types.

## Requirements
1. `WsClient` class wrapping the `ws` package
2. Connect to Hub URL from AgentConfig
3. Auto-reconnect with exponential backoff (1s, 2s, 4s, 8s... max 30s)
4. Send typed messages using the `agentToHubSchema` types
5. Receive and parse messages using `hubToAgentSchema` with Zod
6. Connection state tracking: connecting, connected, disconnected, error
7. Emit typed events for each Hubâ†’Agent message type
8. Send `agent:register` on initial connection
9. Send `agent:heartbeat` every 30s with CPU/mem stats

## Acceptance Criteria
- [ ] Connects to Hub WebSocket at configured URL
- [ ] Sends `agent:register` message on connect
- [ ] Heartbeat sent every 30s with system metrics
- [ ] Auto-reconnects after disconnect (exponential backoff)
- [ ] All incoming messages parsed with Zod (invalid messages logged and skipped)
- [ ] Type-safe `send()` method accepting `AgentToHubMessage`

## Technical Notes
- Use `ws` package (not the browser WebSocket)
- CPU/mem: `os.cpus()` for CPU, `os.freemem()/os.totalmem()` for memory
- Heartbeat should not send if disconnected
- On reconnect, re-send `agent:register`
