# 011-001: Fix shell injection vectors in agent package

## Context
The code review found 4 shell injection vulnerabilities in the agent package where `execSync()` is used with string interpolation instead of `execFileSync()` with argument arrays. These are RCE vectors if Hub messages or config can influence the interpolated values.

## Requirements
1. Replace `execSync` with `execFileSync` in `workspace-provisioner.ts` lines 64 and 93
2. Replace `execSync` with `execFileSync` in `health.ts` line 46 (`getDiskPercent`)
3. Add `isAllowedCommand` allowlist validation for `setupCommands` in `workspace-provisioner.ts` (matching the pattern already used in `container-setup.ts`)
4. Fix unquoted `cwd` in SSH remote command string in `spawn-ssh.ts` line 37

## Acceptance Criteria
- [ ] No `execSync` calls with template string interpolation remain in agent package
- [ ] `workspace-provisioner.ts` uses `execFileSync('du', ['-sb', path])` instead of backtick string
- [ ] `workspace-provisioner.ts` validates `setupCommands` against allowlist before execution
- [ ] `health.ts` uses `execFileSync('df', ['-P', path])` with TypeScript output parsing
- [ ] `spawn-ssh.ts` properly quotes `cwd` in remote command string using shell escaping
- [ ] Type check passes (`pnpm type-check`)
- [ ] Tests pass (`npx vitest run`)
- [ ] Build succeeds

## Technical Notes
**Review findings:** HI-11, HI-12, HI-13, ME-39

Pattern to follow — `container-worktree.ts` already uses `execFileSync` correctly:
```typescript
execFileSync('git', ['clone', '--filter=blob:none', '--', repoUrl, repoDir], execOpts);
```

For `setupCommands` allowlist, reuse `container-setup.ts`'s `ALLOWED_SETUP_PREFIXES` pattern:
```typescript
const ALLOWED_SETUP_PREFIXES = ['npm ', 'pnpm ', 'yarn ', 'pip ', 'cargo ', 'go ', 'make'];
function isAllowedCommand(cmd: string): boolean {
  return ALLOWED_SETUP_PREFIXES.some(p => cmd.startsWith(p));
}
```

For SSH `cwd` quoting: `cwd.replace(/'/g, "'\\''")` wrapped in single quotes.
