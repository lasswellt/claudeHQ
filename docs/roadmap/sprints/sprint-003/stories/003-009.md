# 003-009: Hub dashboard WebSocket handler

## Context
Currently the Hub broadcasts all updates to all dashboard clients. This story adds subscribe/unsubscribe semantics so clients only receive updates for resources they're viewing.

## Requirements
1. Parse dashboard messages with `dashboardToHubSchema`
2. Track subscriptions per client: which sessions, machines, queues they care about
3. On `subscribe` message: add to subscription set, send current state
4. On `unsubscribe` message: remove from set
5. Only forward `session:output` to clients subscribed to that session
6. Broadcast `machine:updated` to clients subscribed to that machine (or all machines)
7. Send initial state snapshot when subscribing

## Acceptance Criteria
- [ ] Dashboard sends `subscribe` for a session, receives its output
- [ ] Dashboard sends `unsubscribe`, stops receiving output
- [ ] Subscribing to all sessions (no id) receives all output
- [ ] Machine updates only sent to interested clients
- [ ] Initial state sent on subscribe
