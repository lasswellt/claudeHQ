# 002-006: Hub agent WebSocket handler

## Context
The Hub's `/ws/agent` endpoint handles all agent connections. It parses messages with Zod, routes them to the appropriate handler, updates the database, and relays output to dashboard subscribers.

## Requirements
1. Parse all incoming messages with `agentToHubSchema`
2. Handle `agent:register` — upsert machine record
3. Handle `agent:heartbeat` — update machine last_seen, CPU/mem
4. Handle `agent:session:started` — insert session record
5. Handle `agent:session:output` — append to recording file, forward to dashboard subscribers
6. Handle `agent:session:ended` — update session record (exit code, ended_at)
7. Handle `agent:recording:upload` — append chunks to JSONL recording file
8. Track connected agents by machineId
9. Detect disconnect → mark machine offline after 60s timeout
10. Send Hub→Agent commands to the correct agent by machineId

## Acceptance Criteria
- [ ] Agent connects and machine appears in `machines` table
- [ ] Heartbeat updates `last_seen` and machine meta
- [ ] Session lifecycle events create/update session records
- [ ] Recording chunks appended to `{recordingsPath}/{sessionId}.jsonl`
- [ ] Hub can send `hub:session:start` to a specific agent
- [ ] Agent disconnect detected, machine marked offline
