# 002-003: Agent CLI commands

## Context
The `chq` CLI is the user-facing interface for managing the agent. It communicates with the daemon process.

## Requirements
1. Use `commander` for CLI framework
2. Commands:
   - `chq agent start` — start the daemon (foreground or background)
   - `chq agent stop` — send SIGTERM to daemon
   - `chq agent status` — show running sessions + connection status
   - `chq run "prompt"` — spawn a session via the daemon
   - `chq run --cwd /path "prompt"` — specify working directory
   - `chq run --flags "..." "prompt"` — custom Claude flags
   - `chq sessions` — list active sessions
   - `chq kill <sessionId>` — kill a specific session
3. For `run`, `sessions`, and `kill`: communicate with daemon via local HTTP or Unix socket

## Acceptance Criteria
- [ ] `chq agent start` launches daemon successfully
- [ ] `chq run "hello world"` spawns a session
- [ ] `chq sessions` lists running sessions with status
- [ ] `chq kill <id>` terminates a session
- [ ] `chq agent status` shows connection state + session count
- [ ] `--help` works for all commands
