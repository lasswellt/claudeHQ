# 002-005: Hub SQLite data access layer

## Context
Sprint 001 created the schema and migrations. This story adds typed query functions for all CRUD operations — the DAL that routes, WS handlers, and the relay will use.

## Requirements
1. Machine operations: `upsertMachine`, `getMachine`, `listMachines`, `updateMachineStatus`
2. Session operations: `insertSession`, `updateSession`, `getSession`, `listSessions` (with filters: machine, status), `updateSessionStatus`
3. Queue operations: `insertQueueTask`, `removeQueueTask`, `listQueueTasks`, `reorderQueue`
4. Notification operations: `insertNotification`, `listNotifications`, `getNotificationConfig`, `updateNotificationConfig`
5. Session event operations: `insertSessionEvent`, `listSessionEvents`
6. All queries use parameterized prepared statements
7. Return typed results matching the shared Zod schemas

## Acceptance Criteria
- [ ] All CRUD operations work with typed inputs and outputs
- [ ] `listSessions({ machineId, status })` filters correctly
- [ ] `upsertMachine` creates or updates based on ID
- [ ] Prepared statements cached (not re-created per call)
- [ ] Transactions used for multi-statement operations (e.g., queue reorder)
- [ ] Tests verify each operation

- `packages/hub/src/dal.ts` (new)
