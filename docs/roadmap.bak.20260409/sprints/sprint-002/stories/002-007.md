# 002-007: Hub REST API — sessions and machines

## Context
The REST API is used by the dashboard and CLI tools to list machines, manage sessions, and send commands. It delegates to the DAL for persistence and the agent handler for command relay.

## Requirements
1. `GET /api/machines` — list registered agents + status
2. `GET /api/machines/:id` — machine detail + active sessions
3. `GET /api/sessions` — list all sessions (filterable: `?machine=X&status=running`)
4. `GET /api/sessions/:id` — session detail
5. `POST /api/sessions` — start new session (validate agent capacity, send `hub:session:start`)
6. `DELETE /api/sessions/:id` — kill running session (send `hub:session:kill`)
7. `POST /api/sessions/:id/input` — send PTY input (send `hub:session:input`)
8. `GET /api/sessions/:id/recording` — stream JSONL recording file
9. Request body validation with Zod
10. Proper HTTP status codes

## Acceptance Criteria
- [ ] All routes respond with correct status codes
- [ ] `POST /api/sessions` validates body, checks agent capacity, relays to agent
- [ ] `DELETE /api/sessions/:id` sends kill command to agent
- [ ] `GET /api/sessions?status=running` filters correctly
- [ ] `GET /api/sessions/:id/recording` streams JSONL file
- [ ] Invalid requests return 400 with error message
