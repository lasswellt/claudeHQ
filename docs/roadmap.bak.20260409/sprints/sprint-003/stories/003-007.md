# 003-007: Session detail page

## Context
The session detail page is the core working view — live terminal output, session metadata, input bar, and action buttons. This is where users interact with running Claude Code sessions.

## Requirements
1. Page at `/sessions/:id` loads session from store (or fetches)
2. TerminalView component showing live output (subscribes to session WS)
3. TerminalInput component below terminal
4. Session metadata panel: prompt, machine, status, duration, cwd, started_at
5. Action buttons: Kill (DELETE /api/sessions/:id), Back
6. Session state indicator: status badge with color + text
7. Handle session not found (404)

## Acceptance Criteria
- [ ] Terminal renders live output for running sessions
- [ ] Input bar sends PTY input
- [ ] Kill button terminates session (with confirmation VDialog)
- [ ] Metadata panel shows all session info
- [ ] Status badge updates in real time via WS
- [ ] 404 handled gracefully

- `packages/dashboard/app/pages/sessions/[id].vue`
- `packages/dashboard/app/components/session/SessionMeta.vue` (new)
- `packages/dashboard/app/components/session/SessionStateIndicator.vue` (new)
