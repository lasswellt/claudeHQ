# 003-006: Overview page

## Context
The landing page shows a grid of machine cards (status, session count, CPU/mem) and a recent session list. This is the first thing users see.

## Requirements
1. Machine cards section: VCard for each machine showing name, status dot, active sessions, max sessions, CPU/mem percentages
2. Session list section: VDataTable with columns — status badge, prompt (truncated), machine, duration, quick actions (terminal link, kill)
3. "New Session" button opening a dialog (VDialog) with machine selector, prompt textarea, cwd input
4. Three states: loading (VSkeletonLoader), empty (VAlert), error (VAlert + retry)
5. Real-time updates via Pinia stores (WebSocket-driven)

## Acceptance Criteria
- [ ] Machine cards show for each registered machine
- [ ] Status dot: green=online, red=offline
- [ ] Session list shows recent sessions sorted by created_at desc
- [ ] "New Session" dialog creates session via POST /api/sessions
- [ ] Loading/empty/error states handled
- [ ] Click session row navigates to /sessions/:id
