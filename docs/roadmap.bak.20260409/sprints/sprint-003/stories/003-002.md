# 003-002: Dashboard WebSocket composable

## Context
The dashboard connects to the Hub via WebSocket to receive real-time updates (session output, machine status, queue changes). This composable manages the connection lifecycle, message parsing, and subscription semantics.

## Requirements
1. `useWebSocket()` composable that connects to `/ws/dashboard`
2. Auto-reconnect with exponential backoff (1s, 2s, 4s... max 30s)
3. Connection state ref: `connecting | connected | disconnected | error`
4. Parse incoming messages with `hubToDashboardSchema` from `@chq/shared`
5. Type-safe `send()` method for `DashboardToHubMessage`
6. `subscribe(resource, id?)` and `unsubscribe(resource, id?)` methods
7. `onMessage(type, handler)` for type-specific event handlers
8. Clean disconnect on composable disposal

## Acceptance Criteria
- [ ] Connects to Hub WebSocket on composable mount
- [ ] Reconnects automatically after disconnect
- [ ] Connection state reflects reality (green dot in UI)
- [ ] Invalid messages logged but don't crash
- [ ] subscribe/unsubscribe sends correct protocol messages
