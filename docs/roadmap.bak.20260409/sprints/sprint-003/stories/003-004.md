# 003-004: Terminal view component

## Context
The core visual component — renders live Claude Code terminal output in the browser using xterm.js with WebGL acceleration and responsive resizing.

## Requirements
1. `TerminalView.vue` component with props: `sessionId`, `readonly`
2. Initialize `@xterm/xterm` Terminal with theme matching Vuetify dark/light
3. Load `@xterm/addon-fit` for responsive resize via ResizeObserver
4. Load `@xterm/addon-webgl` with try/catch (DOM renderer fallback)
5. Subscribe to session output via WebSocket
6. Write received chunks to terminal: `terminal.write(chunk.data)`
7. Dispose terminal in `onUnmounted` (prevent memory leaks)
8. Handle container resize: `fitAddon.fit()` on ResizeObserver callback

## Acceptance Criteria
- [ ] Terminal renders in a container div with correct dimensions
- [ ] WebGL addon loaded (or DOM fallback without error)
- [ ] ResizeObserver triggers fit on container size change
- [ ] Session output appears in terminal in real time (via WS)
- [ ] Terminal properly disposed on component unmount
- [ ] Theme respects dark/light mode

## Technical Notes
- Import from `@xterm/xterm` NOT `xterm` (scoped packages since v5.4)
- WebGL: `try { terminal.loadAddon(new WebglAddon()); } catch { /* DOM fallback */ }`
- CSS: `import '@xterm/xterm/css/xterm.css'`
- ResizeObserver: create in onMounted, disconnect in onUnmounted
- Debounce fit() calls to avoid rapid-fire during resize drag
