# 003-008: Machine detail page

## Context
The machine detail page shows a single agent's status, active sessions, completed session history, and basic health info.

## Requirements
1. Page at `/machines/:id` loads machine from store
2. Machine info card: name, status, OS, max sessions, last seen
3. Active sessions list (running + queued)
4. Recent completed sessions list
5. Start session button (opens NewSessionModal pre-filled with this machine)
6. Handle machine not found

## Acceptance Criteria

- [ ] Active and completed sessions listed
- [ ] Start session pre-selects this machine
- [ ] Real-time status updates via WS
- [ ] 404 handled
