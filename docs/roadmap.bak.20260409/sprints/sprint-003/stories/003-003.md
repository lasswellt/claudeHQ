# 003-003: Pinia stores for sessions and machines

## Context
Pinia stores are the reactive state layer between the WebSocket composable and the Vue components. They fetch initial data via REST and receive real-time updates via WebSocket.

## Requirements
1. `useSessionsStore()` — setup syntax, returns sessions, loading, error, activeSessions computed, fetchSessions action, WS update handler
2. `useMachinesStore()` — setup syntax, returns machines, loading, error, onlineMachines computed, fetchMachines action, WS update handler
3. Both stores handle three states: loading, data, error
4. WS handlers: `session:updated` updates the session in the store, `machine:updated` updates the machine
5. Initial data loaded via `fetch('/api/sessions')` and `fetch('/api/machines')`

## Acceptance Criteria
- [ ] `useSessionsStore().sessions` reactive to REST fetch and WS updates
- [ ] `useMachinesStore().machines` reactive to REST fetch and WS updates
- [ ] `activeSessions` computed filters running sessions
- [ ] `onlineMachines` computed filters online machines
- [ ] Error state populated on fetch failure
- [ ] Setup syntax follows project rules

## Files Likely Affected
- `packages/dashboard/app/stores/sessions.ts` (new)
- `packages/dashboard/app/stores/machines.ts` (new)
