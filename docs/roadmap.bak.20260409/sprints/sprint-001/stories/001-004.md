# 001-004: Vitest setup with shared configuration

## Context
Vitest is the test runner for all packages. A shared base config ensures consistent testing patterns across the monorepo.

## Requirements
1. Install `vitest` as root devDependency
2. Create `vitest.config.ts` at root (or `vitest.workspace.ts` for workspace mode)
3. Configure per-package test paths
4. Add a sample test in `packages/shared` to verify setup
5. Ensure TypeScript paths resolve in tests

## Acceptance Criteria
- [ ] `npx vitest run` succeeds from root
- [ ] `pnpm --filter @chq/shared test` runs only shared tests
- [ ] Sample test passes: import from shared, assert type correctness
- [ ] Coverage reporting works (optional, can be enabled later)

## Technical Notes
- Vitest workspace mode: `vitest.workspace.ts` with `['packages/*']` glob
- Each package can override with local `vitest.config.ts` if needed
- Use `@vitest/coverage-v8` for coverage (install later)
- Test file convention: `*.test.ts` colocated with source or in `__tests__/` dir

## Files Likely Affected
- `vitest.workspace.ts` (or `vitest.config.ts`)
- `packages/shared/vitest.config.ts` (optional override)
- `packages/shared/src/__tests__/index.test.ts` (sample test)
