# 001-009: WebSocket protocol ‚Äî Hub‚ÜíDashboard messages

## Context
Dashboard receives real-time updates from the Hub via WebSocket. These messages drive the entire live UI. Subscribe/unsubscribe semantics are included for selective updates.

## Requirements
1. Hub‚ÜíDashboard messages:
   - `session:output` ‚Äî sessionId, chunks: Array<{ts, data}>
   - `session:updated` ‚Äî session: SessionRecord
   - `machine:updated` ‚Äî machine: MachineRecord
   - `queue:updated` ‚Äî machineId, queue: QueueTask[]
   - `notification` ‚Äî notification: NotificationRecord
2. Dashboard‚ÜíHub messages:
   - `subscribe` ‚Äî resource: 'session'|'machine'|'queue', id?: string
   - `unsubscribe` ‚Äî resource: 'session'|'machine'|'queue', id?: string
3. Discriminated unions for each direction

## Acceptance Criteria
- [ ] `hubToDashboardSchema.parse(msg)` produces typed union
- [ ] `dashboardToHubSchema.parse(msg)` produces typed union
- [ ] Subscribe/unsubscribe messages parse with correct resource types
- [ ] All record references use the types from 001-007

## Technical Notes
- `session:output` reuses the same chunk format as `agent:session:output`
- `session:updated` wraps a full `SessionRecord` (not a partial update)
- Subscribe semantics: `id` is optional ‚Äî omit to subscribe to all of a resource type
