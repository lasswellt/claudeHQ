# 001-011: PTY session class

## Context
The core of the Agent — a class that wraps `node-pty` to manage a single Claude Code PTY session. It handles spawning, output streaming, input delivery, and graceful/forced termination.

## Requirements
1. `PtySession` class with:
   - Constructor: `{ id, command, args, cwd, env, onOutput, onExit }`
   - `spawn()` method: creates node-pty instance, starts output streaming
   - `write(data: string)` method: sends input to PTY stdin
   - `kill()` method: SIGTERM, then SIGKILL after 5s timeout
   - `resize(cols, rows)` method: resize the PTY
   - State tracking: `spawning | running | completed | failed`
   - Exit code and signal capture
2. Output streaming: buffer PTY onData events into `{ ts: number, data: string }` chunks
3. Claude session ID extraction: parse PTY output for session ID (from stream-json or file)
4. Session naming: spawn with `-n chq:<machineId>:<sessionId>` flag
5. Error handling: spawn failures, unexpected exits, PTY errors

## Acceptance Criteria
- [ ] `new PtySession({...}).spawn()` creates a running PTY process
- [ ] Output chunks emitted via callback with millisecond timestamps
- [ ] `session.write("yes\n")` delivers input to the PTY
- [ ] `session.kill()` terminates the process (SIGTERM then SIGKILL)
- [ ] State transitions: spawning → running → completed/failed
- [ ] Exit code captured in `session.exitCode`
- [ ] PTY disposed on completion (no resource leaks)
- [ ] Type check passes, tests pass

## Technical Notes
- `node-pty` API: `spawn(file, args, options)` returns `IPty`
- `pty.onData(callback)` for output, `pty.write(data)` for input, `pty.kill()` for termination
- `pty.onExit(({ exitCode, signal }) => {...})` for process end
- Timestamp: `Date.now()` relative to session start (first chunk ts=0)
- Claude Code spawn: `claude -p "${prompt}" --dangerously-skip-permissions -n "chq:${machineId}:${sessionId}"`
- Default flags from agent config, overridable per session
- node-pty is NOT thread-safe — all operations on the main thread
