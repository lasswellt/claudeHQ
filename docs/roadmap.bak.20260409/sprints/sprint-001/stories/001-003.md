# 001-003: Turborepo pipeline and build scripts

## Context
Turborepo orchestrates builds, tests, and linting across the monorepo with dependency-aware caching. The pipeline definition determines build order and parallelism.

## Requirements
1. Install `turbo` as a root devDependency
2. Create `turbo.json` with pipelines: `build`, `test`, `lint`, `type-check`
3. `build` pipeline: depends on `^build` (build deps first), outputs `dist/**`
4. `test` pipeline: depends on `build`, no caching (tests should always run)
5. `lint` pipeline: no deps, cacheable
6. `type-check` pipeline: no deps, cacheable
7. Add root scripts: `build`, `test`, `lint`, `type-check`, `dev`
8. Configure shared package build (tsup or tsc) to produce dist/
9. Verify `turbo build` builds shared → agent, hub in correct order
10. Verify `turbo prune @chq/hub --docker` produces correct output

## Acceptance Criteria
- [ ] `pnpm turbo build` succeeds, building shared first
- [ ] `pnpm turbo test` runs vitest across all packages
- [ ] `pnpm turbo lint` runs ESLint across all packages
- [ ] `turbo prune @chq/hub --docker` generates `out/json/` and `out/full/`
- [ ] `turbo prune @chq/agent --docker` generates correct subset
- [ ] Build outputs cached on second run (shows "FULL TURBO")

## Technical Notes
- Use `tsup` for shared package build: `tsup src/index.ts --format esm,cjs --dts`
- Agent and hub: `tsup src/index.ts --format esm` or `tsc`
- turbo.json pipeline `dependsOn: ["^build"]` means "build my workspace deps first"
- Dashboard uses `nuxt build` / `nuxt generate` (configured separately)

## Files Likely Affected
- `turbo.json`
- `package.json` (root scripts)
- `packages/shared/package.json` (build script, tsup config)
- `packages/agent/package.json` (build script)
- `packages/hub/package.json` (build script)
