# 001-015: Shared protocol tests

## Context
The protocol schemas are the foundation everything builds on. Thorough tests ensure schema changes are caught early and that all packages can rely on consistent message parsing.

## Requirements
1. Test each Agent→Hub message schema: valid data parses, invalid data throws ZodError
2. Test each Hub→Agent message schema: same pattern
3. Test Hub→Dashboard and Dashboard→Hub schemas
4. Test discriminated union parsing: correct type narrowing
5. Test config schemas: valid config, missing required fields, default values
6. Test record types: valid records, type inference

## Acceptance Criteria
- [ ] 100% of message types have at least one valid and one invalid test case
- [ ] Discriminated union tests verify correct type narrowing after parse
- [ ] Config schema tests verify default value application
- [ ] All tests pass: `pnpm --filter @chq/shared test`

## Technical Notes
- Use `expect(() => schema.parse(invalid)).toThrow(ZodError)` pattern
- Test discriminated unions: `const msg = agentToHubSchema.parse({type: 'agent:register', ...}); if (msg.type === 'agent:register') { expect(msg.machineId).toBe(...) }`
- Test defaults: `const config = hubConfigSchema.parse({}); expect(config.port).toBe(7700)`
