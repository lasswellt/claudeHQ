# 001-001: Initialize pnpm workspace with 4 packages

## Context
This is the very first story — it creates the monorepo skeleton that everything else builds on. Without the workspace, no package can be built or tested.

## Requirements
1. Create `pnpm-workspace.yaml` with `packages/*` glob
2. Create root `package.json` with workspace scripts and `packageManager` field for corepack
3. Create `packages/shared/package.json` as `@chq/shared`
4. Create `packages/agent/package.json` as `@chq/agent` with dependency on `@chq/shared`
5. Create `packages/hub/package.json` as `@chq/hub` with dependency on `@chq/shared`
6. Create `packages/dashboard/package.json` as `@chq/dashboard` with dependency on `@chq/shared`
7. Each package should have a minimal `src/index.ts` exporting a placeholder
8. Run `pnpm install` to verify workspace resolution

## Acceptance Criteria
- [ ] `pnpm install` succeeds with all 4 packages resolved
- [ ] `pnpm ls --depth 0` shows all workspace packages
- [ ] Each package can import from `@chq/shared` (workspace protocol link)
- [ ] `packageManager` field set in root package.json (e.g., `pnpm@9.x.x`)
- [ ] Node.js engines field set to `>=20` in root package.json

## Technical Notes
- Use `workspace:*` protocol for inter-package dependencies
- Package naming: `@chq/shared`, `@chq/agent`, `@chq/hub`, `@chq/dashboard`
- pnpm-workspace.yaml: `packages: ['packages/*']`
- Set `"private": true` on root and all packages (not publishing to npm)

## Files Likely Affected
- `package.json` (root)
- `pnpm-workspace.yaml`
- `packages/shared/package.json`
- `packages/agent/package.json`
- `packages/hub/package.json`
- `packages/dashboard/package.json`
- `packages/*/src/index.ts` (placeholder exports)
