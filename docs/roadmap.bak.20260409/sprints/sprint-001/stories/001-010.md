# 001-010: Config schemas

## Context
Both Agent and Hub load configuration from files and environment variables. Zod schemas provide validation with clear error messages when config is wrong.

## Requirements
1. Agent config schema (`AgentConfig`):
   - machineId, displayName, hubUrl, claudeBinary, defaultFlags, defaultCwd
   - maxConcurrentSessions, recordingChunkIntervalMs, recordingUploadIntervalMs
2. Hub config schema (`HubConfig`):
   - port, databasePath, recordingsPath, logLevel
3. Shared config utilities: `loadConfig<T>(schema, filePath, envPrefix?)` that merges file + env

## Acceptance Criteria
- [ ] `AgentConfig` and `HubConfig` schemas exported from `@chq/shared`
- [ ] Invalid config produces clear error: "machineId is required"
- [ ] Environment variable override works: `CHQ_AGENT_MACHINE_ID` overrides `machineId`
- [ ] Default values applied for optional fields

## Technical Notes
- Use `z.object({...}).default({...})` for defaults
- Env prefix convention: `CHQ_AGENT_` for agent, `CHQ_HUB_` for hub
- `loadConfig` reads JSON file, overlays env vars, validates with schema
- Consider `zod-config` or custom implementation
